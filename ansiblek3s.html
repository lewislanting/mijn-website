AnsibleK3s

Airgapped install of K3s through ansible

Beschrijving

In dit project voerde ik een air-gapped installatie uit van K3s, een lightweight Kubernetes-distributie, op meerdere machines binnen mijn homelab. De installatie werd volledig geautomatiseerd via Ansible, een configuratie- en automatiseringstool gebaseerd op Python. Het doel van dit project was het opzetten van een Kubernetes-cluster zonder internettoegang op de doelmachines, wat in realistische omgevingen (zoals gesloten netwerken of industriële systemen) vaak voorkomt. De complexiteit van een air-gapped setup ligt in het vooraf ophalen, lokaal hosten en beschikbaar stellen van alle benodigde bestanden: K3s binaries, container images, Ansible roles en dependencies. Hierdoor kreeg ik een dieper inzicht in de werking van zowel K3s als Ansible en de onderliggende Python-bibliotheken.

Situatieschets

  • Air-gapped betekent: geen internettoegang op de target nodes → dus alle K3s-binaries, images en dependencies moeten vooraf manueel of via een proxy worden voorbereid.
  • Gebruik van Ansible om alle taken te automatiseren: van systeemvoorbereiding tot het uitrollen van K3s.
  • Ik onderzocht hoe ik de Python-omgeving kon voorbereiden zonder externe repositories, met behulp van tools als pip download, venv, en apt-offline.

Eindoplevering

  • Een volledig functionerend K3s-cluster, geautomatiseerd uitgerold via Ansible op machines zonder internetverbinding.
  • Een lokale repository met alle benodigde: binaries, scripts, images, Python dependencies.
  • Ansible playbooks die in logische stappen zorgen voor: Installatie, configuratie, deployment, laden van images.

Reflectie

Dit project vergde een grondige voorbereiding en bracht me in contact met reële uitdagingen die je in beveiligde of industriële netwerken tegenkomt. Door te werken in een air-gapped context, werd ik gedwongen om elke afhankelijkheid te analyseren en lokaal beschikbaar te maken — van container images tot Python libraries. Ik kreeg een dieper begrip van hoe Ansible op Python is gebouwd en hoe het remote taken uitvoert. Vooral het feit dat Python op de target machines een harde vereiste is voor Ansible werd duidelijk, evenals het belang van versiebeheer en consistentie in bibliotheken. Dit project leerde mij het belang van automatisering én voorbereiding. Een fout in de afhankelijkheidsketen betekent volledige herinstallatie of troubleshooting zonder internettoegang. De combinatie van infrastructuurkennis, scripting, containerbeheer en Python package management maakt dit een van de meest technische en leerzame projecten die ik tot nu toe heb uitgevoerd.
